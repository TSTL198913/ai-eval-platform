#### 问题1：未校准评估器是否允许执行？

**当前实现**：
```python
# src/domain/adaptive_calibration.py:328
if not calibration_status.get("calibration_score"):
    return PreExecutionCheck(
        can_proceed=True,  # ✅ 允许执行
        status=CalibrationStatus.NOT_CALIBRATED,
        message="评估器尚未校准，建议先在黄金数据集上校准",
        warnings=["评估结果可能不可靠"]
    )
```

**业务场景**：
- 用户首次使用新评估器
- 评估器尚未在黄金数据集上校准
- 用户发起评估请求

**产品经理决策点**：

| 选项 | 说明 | 用户影响 |
|------|------|---------|
| **A. 允许执行** | 未校准也可评估，给出警告 | 用户可立即使用，但结果可能不可靠 |
| **B. 拒绝执行** | 未校准必须先校准 | 用户必须先校准，结果可靠但流程复杂 |
| **C. 分级执行** | 低风险场景允许，高风险拒绝 | 平衡灵活性和可靠性 |

**建议**：选择A，允许执行但给出警告，保持灵活性。

**确认问题**：
- ❓ 未校准评估器是否允许执行？
- ❓ 如果允许，警告信息是否足够清晰？
- ❓ 是否需要强制校准的场景？

### 🟡 P1：阈值设置

#### 问题4：校准阈值5%是否合理？

**当前实现**：
```python
# src/domain/adaptive_calibration.py:108
default_threshold: float = 5.0  # 默认偏差阈值(%)

# src/domain/evaluator_version.py:195
baseline_score = 95.0  # 默认基准分数
deviation_pct = abs(calibration_score - baseline_score) / baseline_score * 100
is_calibrated = deviation_pct <= threshold  # 默认5%
```

**业务场景**：
- 评估器校准后，偏差5%以内算通过
- 基准分数95分（专家平均分）
- 评估器分数90分时，偏差5.26%，校准失败

**产品经理决策点**：

| 选项 | 说明 | 用户影响 |
|------|------|---------|
| **A. 5%阈值** | 严格校准标准 | 评估器质量高，但可能频繁校准失败 |
| **B. 10%阈值** | 宽松校准标准 | 评估器更容易通过，但质量可能降低 |
| **C. 可配置阈值** | 用户可设置阈值 | 灵活性高，但用户可能不理解 |

**建议**：选择C，可配置阈值，保持灵活性。

**确认问题**：
- ❓ 校准阈值5%是否合理？
- ❓ 是否需要支持用户自定义阈值？
- ❓ 不同评估器是否需要不同阈值？

## 三、用户使用流程确认

### 流程1：首次使用评估器

```
1. 用户选择评估器
2. 系统检查校准状态
   ├─ 已校准 → 直接执行评估
   └─ 未校准 → 显示警告，允许执行 ❓
3. 用户确认执行
4. 系统返回评估结果
5. 用户查看结果
```

**确认问题**：
- ❓ 步骤2是否应该拒绝执行？
- ❓ 警告信息是否足够清晰？

### 流程2：校正评估结果

```
1. 用户查看评估结果
2. 用户认为评分不准确
3. 用户发起校正请求
4. 系统记录校正 ❓（是否需要审核？）
5. 校正生效，更新黄金数据集
6. Few-shot示例更新
```

**确认问题**：
- ❓ 步骤4是否需要审核流程？
- ❓ 校正后是否需要通知相关人员？

### 流程3：漂移检测

```
1. 系统定期检测漂移
2. 发现漂移
3. 系统通知用户 ❓（是否需要通知？）
4. 用户查看漂移报告
5. 用户决定是否重新校准
```

**确认问题**：
- ❓ 步骤3是否需要自动通知？
- ❓ 漂移阈值是否合理？
