# AI-Eval-Pro 部署测试策略

## 一、测试目标

验证部署到服务器的 AI-Eval-Pro 系统能否正常运行，确保：
1. API服务健康可用
2. 前端与后端集成正常
3. 认证授权功能正常
4. 评估器核心功能可用
5. 监控告警系统正常

## 五、测试执行命令

```bash

pytest tests/deployment/test_health.py -v

# 仅运行认证测试

# 运行评估器功能测试

# 运行监控验证测试

pytest tests/deployment/ -v --html=deployment_test_report.html
```

## 七、风险点

| 风险等级 | 风险描述 | 应对措施 |
|----------|----------|----------|
| P0 | API服务不健康导致所有测试失败 | 先修复Dockerfile添加curl，重建镜像 |
| P0 | 认证失败导致无法访问受保护资源 | 验证ADMIN_PASSWORD环境变量设置正确 |
| P1 | 数据库连接失败 | 验证PostgreSQL服务正常运行 |
| P1 | Redis连接失败 | 验证Redis服务正常运行 |
| P1 | 评估器依赖缺失 | 检查requirements.txt是否完整 |
| P2 | CORS配置错误 | 验证CORS_ALLOWED_ORIGINS包含前端IP |
