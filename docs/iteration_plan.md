## 二、需求分析

### 2.1 核心问题

当前后端已有 **81个API端点**，但部分核心功能未集成到Routes，导致前端页面存在功能缺失：

| 前端页面 | 已有Routes | 缺失功能 | 优先级 |
|----------|------------|----------|--------|
| 评估首页 | evaluation_routes.py (4) | Benchmark基准测试 | P0 |
| 模型管理页 | model_routes.py (3) | 模型版本管理 | P0 |
| 数据集页 | dataset_routes.py (2) | 黄金数据集管理 | P0 |
| 模型对比页 | model_comparison.py (4) | A/B测试 | P0 |
| 评估器管理页 | evaluator_routes.py (5) | 评估器版本管理 | P1 |

### 2.2 权限体系需求

当前系统缺少细粒度RBAC权限控制，需要扩展角色和权限定义：

| 需求 | 说明 |
|------|------|
| 角色扩展 | 新增标注员、安全管理员、质量管理员、财务管理员等角色 |
| 权限扩展 | 新增安全测试、质量门禁、成本治理、元评估等权限 |
| 权限装饰器 | 提供@require_permissions、@require_roles等装饰器 |

## 三、技术方案

### 3.1 RBAC权限设计

#### 角色定义

```python
class Role(Enum):
    # 基础角色
    GUEST = "guest"              # 访客（只读）
    USER = "user"                # 普通用户（基础评测）
    ANNOTATOR = "annotator"      # 标注员（人工标注）

    # 高级角色
    PREMIUM = "premium"          # 高级用户（模型对比）
    ENTERPRISE = "enterprise"    # 企业用户（完整功能）

    # 管理角色
    ADMIN = "admin"              # 系统管理员
    SECURITY_ADMIN = "security_admin"      # 安全管理员
    QUALITY_ADMIN = "quality_admin"        # 质量管理员
    FINANCE_ADMIN = "finance_admin"        # 财务管理员
    DEVOPS = "devops"                        # 运维工程师

    # 超级管理员
    SUPER_ADMIN = "super_admin"    # 超级管理员（所有权限）
```

#### 权限定义（核心权限）

| 权限 | 描述 | 适用角色 |
|------|------|----------|
| `CREATE_AB_TEST` | 创建A/B测试 | USER+ |
| `RUN_BENCHMARK` | 运行基准测试 | USER+ |
| `MANAGE_GOLDEN_DATASET` | 管理黄金数据集 | ENTERPRISE+ |
| `MANAGE_MODEL_VERSIONS` | 管理模型版本 | ADMIN+ |
| `RUN_SECURITY_TEST` | 运行安全测试 | SECURITY_ADMIN+ |
| `RUN_QUALITY_GATE` | 运行质量门禁 | QUALITY_ADMIN+ |
| `MANAGE_BUDGET` | 管理预算 | FINANCE_ADMIN+ |

#### 权限装饰器

```python
@require_permissions(Permission.CREATE_AB_TEST)
@require_roles(Role.ADMIN, Role.SUPER_ADMIN)
@require_admin
@require_super_admin
```

### 3.2 API设计规范

#### 统一响应格式

```json
{
    "code": 0,
    "message": "success",
    "data": {...},
    "timestamp": "2026-06-24T10:00:00Z"
}
```

#### 错误响应格式

```json
{
    "code": 403,
    "message": "权限不足，需要权限: manage_models",
    "data": null
}
```

## 五、测试计划

### 5.1 测试策略

| 测试类型 | 覆盖范围 | 优先级 |
|----------|----------|--------|
| **单元测试** | RBAC权限装饰器、核心API逻辑 | P0 |
| **集成测试** | API端点与前端接口契约 | P0 |
| **安全测试** | 权限绕过、越权访问 | P0 |
| **功能测试** | A/B测试、Benchmark完整流程 | P1 |
| **性能测试** | API响应时间、并发处理 | P2 |

### 5.2 测试用例设计原则

按照项目测试核心哲学：
1. **正向测试**：正常请求流程验证
2. **负向测试**：无权限、参数错误、资源不存在
3. **边界测试**：空数据、超大数据、特殊字符
4. **异常测试**：网络超时、数据库异常、依赖服务异常
5. **依赖测试**：权限继承、角色切换、级联操作

### 5.3 关键测试场景

#### 权限测试场景

| 场景 | 描述 | 预期结果 |
|------|------|----------|
| 普通用户访问管理API | user角色调用/admin/users | 返回403错误 |
| 管理员越权访问安全API | admin角色调用/security/scan | 返回403错误 |
| 超级管理员访问所有API | super_admin角色调用任意API | 返回200成功 |
| 权限继承验证 | enterprise角色调用CREATE_AB_TEST | 返回200成功 |

#### API测试场景

| 场景 | 描述 | 预期结果 |
|------|------|----------|
| 创建A/B测试 | 正常参数创建测试 | 返回测试ID和状态 |
| 创建测试参数缺失 | 缺少model_b参数 | 返回400错误 |
| 运行不存在的基准测试 | 运行不存在的dataset_id | 返回404错误 |
| 导出空数据集 | 导出无样本的数据集 | 返回空列表 |
